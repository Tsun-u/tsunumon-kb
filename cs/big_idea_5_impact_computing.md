# Big Idea 5: Impact of Computing (IOC)

**Exam Weight:** 21-26%

## Overview

Computing technology shapes every aspect of society, and people's daily lives are increasingly intertwined with computing in ways that are easy to overlook. Studying computing helps students think critically about the potential benefits, possible harms, and ethical considerations that accompany any computing innovation.

## Key Topics

### Intended and Unintended Effects of Computing Innovations

Computing innovations don't always behave the way their designers expected.

- People build computing innovations to solve problems or pursue interests
- A single computing innovation can produce both expected effects and unexpected ones
- The same effect can be a benefit for some people or groups and a harm for others
- Advances in computing have also sparked new creative possibilities in music, art, science, and mathematics
- No one can predict every consequence of an innovation at the design stage

### Effects That Go Beyond the Original Purpose

- Computing innovations can end up being used in ways the developers never anticipated
- Responsible developers actively try to think through possible unintended uses and effects during the design process
- Some innovations have evolved far beyond their original scope (social media platforms are a well-known example)
- Machine learning and artificial intelligence have introduced new ethical questions around decision-making and bias

### Privacy Risks from Personal Data

Collecting and storing personal data in the digital age carries real privacy risks:

- **Personally identifiable information (PII)** includes name, date of birth, national ID number, address, email, and phone number
- PII can be exploited for identity theft, stalking, or other privacy violations
- Programs can collect PII through cookies, browsing history, location tracking, and social media activity
- Data collection often happens without the user's full awareness or informed understanding
- GPS devices, smart devices, and web browsers can all log records of user activity

### Intellectual Property and Legal-Ethical Issues

- Work created on a computer belongs to the creator or the organization they work for
- The ease of copying and rapidly distributing digital information makes protecting intellectual property more complicated
- Legal mechanisms for protecting intellectual property include: copyright, patents, and trademarks
- **Open source** licenses and **Creative Commons** licenses allow others to share and adapt work under specific conditions
- Using someone else's copyrighted work requires permission or must fall within fair use provisions
- The accessibility of digital content raises the issue of the **digital divide** — unequal access to technology across different population groups

### How Computing Enables Large-Scale Collaborative Problem Solving

- Broad access to information and open data lets more people participate in identifying problems and proposing solutions
- **Citizen science** invites members of the public to contribute data and analysis to scientific research
- **Crowdsourcing** aggregates the distributed knowledge and resources of a large number of contributors
- Using programs to analyze massive datasets and run simulations has fundamentally changed the way scientific research works

### Personal Security and Authentication Risks

#### Security Threats to Stored Data

- PII and other personal data can be targets of phishing attacks and identity theft
- Authentication mechanisms (passwords, multi-factor authentication) help protect data security
- Strong passwords are long, complex, and unique to each account
- **Multi-factor authentication (MFA)** combines two or more distinct types of identity verification
- **Encryption** is the technique of encoding data to prevent unauthorized access
- Symmetric encryption uses the same key for both encryption and decryption; asymmetric encryption uses a key pair — one public, one private

#### How Computing Resources Can Be Protected and Misused

- **Computer viruses and malware** are programs designed to damage systems or gain unauthorized access
- **Phishing** attacks impersonate a trusted source to trick users into handing over personal information
- **Keyloggers** silently record every keystroke a user makes, potentially capturing passwords and other sensitive data
- Malicious wireless access points can intercept data traveling over a network
- Software vulnerabilities can be exploited by attackers to break into systems

#### Pathways to Unauthorized Access

- Unauthorized access can be achieved through security vulnerabilities, social engineering, or brute-force attacks
- Software patches and updates help close known security holes
- Good security habits include: using strong passwords, keeping software up to date, and staying alert to phishing attempts

## Key Terms

| Term | Definition |
|------|------------|
| PII | Personally identifiable information — data that can identify a specific individual (name, ID number, address, etc.) |
| Digital divide | Unequal access to technology based on demographic characteristics or geographic location |
| Intellectual property | Legally protected mental creations (software, music, writing, etc.) |
| Copyright | A creator's legal right to control how their work is used |
| Open source | Software whose source code is publicly available and free to use and modify |
| Creative Commons | A licensing system that lets creators specify the terms under which their work may be used |
| Fair use | Limited use of copyrighted material without permission under specific conditions (such as education) |
| Phishing | A scam that impersonates a trusted source to steal sensitive information |
| Malware | Software designed to damage or gain unauthorized access to a computer system |
| Encryption | The technique of encoding data so unauthorized parties cannot read it |
| Symmetric encryption | Encryption where the same key is used for both encrypting and decrypting |
| Asymmetric encryption | Encryption using a public key to encrypt and a private key to decrypt |
| Multi-factor authentication | Using two or more verification methods to confirm someone's identity |
| Cookie | A small data file stored on a user's device by a website to track activity |
| Crowdsourcing | Gathering ideas, resources, or solutions from a large number of contributors online |
| Citizen science | A model in which members of the public contribute to scientific research |
| Bias | A systematic error in data or an algorithm that produces unfair or skewed outcomes |

## Common Misconceptions

1. **Computing innovations are always beneficial** — Every innovation can have both positive and negative effects; the same technology can help some people while harming others.
2. **Privacy settings fully protect personal data** — Privacy settings reduce exposure but do not guarantee complete protection. Data can still be collected, shared, or leaked.
3. **Encryption makes data absolutely secure** — Encryption greatly improves security, but it is not unbreakable. Weak keys, implementation flaws, or social engineering attacks can all compromise encrypted data.
4. **The digital divide is only about money** — The digital divide also involves geographic location, age, education level, disability, and language barriers.
5. **Open source means no rules** — Open source licenses still have terms. Some require attribution; some require that modified versions carry the same license.
6. **Being anonymous online means true anonymity** — IP addresses, metadata, and digital footprints can often be traced back to a specific individual.
7. **AI and algorithms are free of bias** — Algorithms can reflect and amplify the biases present in training data or embedded in their design.
8. **Only hackers cause data breaches** — Human error — weak passwords, clicking phishing links, misconfigured systems — is one of the leading causes of data breaches.
