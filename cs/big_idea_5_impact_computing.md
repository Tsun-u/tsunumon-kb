# Big Idea 5: Impact of Computing (IOC)

**Exam Weight:** 21-26%

## Overview

Computing affects many aspects of society. People's lives are increasingly intertwined with computing. The study of computing enables students to think critically about the potential benefits, harms, and ethical considerations of computing innovations.

## Enduring Understandings and Learning Objectives

### IOC-1: While computing innovations are typically designed to achieve a specific purpose, they may have unintended consequences

#### IOC-1.A: Explain how an effect of a computing innovation can be both beneficial and harmful
- **Essential Knowledge:**
  - People create computing innovations to solve problems or pursue interests
  - A computing innovation can have both intended and unintended effects
  - A single effect can be viewed as both beneficial and harmful by different people or groups
  - Advances in computing have generated and increased creativity in other fields (music, art, science, mathematics)
  - Not every effect of a computing innovation is anticipated in advance

#### IOC-1.B: Explain how a computing innovation can have an impact beyond its intended purpose
- **Essential Knowledge:**
  - Computing innovations can be used in ways that their creators had not originally intended
  - Responsible programmers try to consider unintended uses and effects when designing programs
  - Some innovations evolve beyond their original scope (e.g., social media platforms)
  - Machine learning and AI raise new ethical considerations about decision-making and bias

#### IOC-1.C: Describe the risks to privacy from collecting and storing personal data on a computer system
- **Essential Knowledge:**
  - Personally identifiable information (PII) includes name, date of birth, Social Security number, address, email, phone number
  - PII can be used to steal someone's identity, stalk them, or compromise their privacy
  - Programs can collect PII through cookies, browsing history, location tracking, and social media
  - The collection of PII often occurs without the user's knowledge or full understanding
  - Technologies like GPS, smart devices, and web browsers can record users' activities

#### IOC-1.D: Explain how computing innovations can raise legal and ethical concerns
- **Essential Knowledge:**
  - Material created on a computer is the intellectual property of the creator or organization
  - The ease of access and distribution of digitized information raises intellectual property concerns
  - Measures that protect intellectual property include copyright, patents, and trademarks
  - Open source and Creative Commons licenses allow sharing and modification of works
  - The use of copyrighted material must be done with permission or under fair use provisions
  - Access to digital content raises the issue of the digital divide (unequal access based on demographics)

#### IOC-1.E: Explain how people participate in problem-solving processes at scale
- **Essential Knowledge:**
  - Widespread access to information and public data facilitates identifying problems and solutions
  - Citizen science allows individuals to contribute data and analysis to scientific research
  - Crowdsourcing allows distributed solutions and ideas from many contributors
  - Science has been affected by the use of programs to analyze large data sets and run simulations

#### IOC-1.F: Explain how the use of computing can raise legal and ethical concerns
- **Essential Knowledge:**
  - Computing raises questions about intellectual property, privacy, and data security
  - Digital data can be copied, shared, and distributed easily
  - Programs can be used maliciously for phishing, identity theft, or unauthorized data collection

### IOC-2: The use of computing innovations may involve risks to personal safety and identity

#### IOC-2.A: Describe the risks to privacy from collecting and storing personal data on a computer system
- **Essential Knowledge:**
  - PII and other personal data can be a target for phishing and identity theft
  - Authentication measures (passwords, multi-factor authentication) help protect data
  - Strong passwords are long, complex, and unique
  - Multi-factor authentication uses two or more different types of verification
  - Encryption is the process of encoding data to prevent unauthorized access
  - Symmetric encryption uses one key; asymmetric encryption uses a public-private key pair

#### IOC-2.B: Explain how computing resources can be protected and can be misused
- **Essential Knowledge:**
  - Computer viruses and malware are programs designed to damage or gain unauthorized access to systems
  - A phishing attack tricks users into providing personal information
  - Keyloggers capture what a user types, potentially capturing passwords and other sensitive information
  - Rogue access points can intercept data on a network
  - Software vulnerabilities can be exploited by attackers

#### IOC-2.C: Explain how unauthorized access to computing resources is gained
- **Essential Knowledge:**
  - Unauthorized access can be gained through security vulnerabilities, social engineering, or brute force
  - Software patches and updates help fix known vulnerabilities
  - Users should practice good security habits: strong passwords, software updates, awareness of phishing

## Key Concepts and Terminology

| Term | Definition |
|------|-----------|
| PII | Personally Identifiable Information (name, SSN, address, etc.) |
| Digital divide | Unequal access to technology based on demographics or geography |
| Intellectual property | Creations of the mind (software, music, writing) protected by law |
| Copyright | Legal right of a creator to control use of their work |
| Open source | Software whose source code is freely available to use and modify |
| Creative Commons | Licensing system that allows creators to specify how their work can be used |
| Fair use | Limited use of copyrighted material without permission for purposes like education |
| Phishing | Fraudulent attempt to obtain sensitive information by impersonation |
| Malware | Software designed to harm or exploit computer systems |
| Encryption | Encoding data so only authorized parties can read it |
| Symmetric encryption | Uses the same key for encryption and decryption |
| Asymmetric encryption | Uses a public key for encryption and a private key for decryption |
| Multi-factor authentication | Using two or more verification methods to access an account |
| Cookie | Small file stored by a website on a user's computer to track activity |
| Crowdsourcing | Obtaining contributions from many people, often via the internet |
| Citizen science | Public participation in scientific research |
| Bias | Systematic errors in data or algorithms that produce unfair results |

## Common Misconceptions

1. **Computing innovations are always beneficial** - Every innovation has potential for both positive and negative impacts. The same technology can help and harm.
2. **Privacy settings fully protect personal data** - Privacy settings reduce exposure but do not guarantee protection. Data can still be collected, shared, or breached.
3. **Encryption makes data completely safe** - Encryption significantly improves security but is not foolproof. Weak keys, implementation flaws, or social engineering can still compromise data.
4. **The digital divide is only about money** - It also involves geography, age, education, disability access, and language barriers.
5. **Open source means no rules** - Open source licenses still have terms. Some require attribution, sharing modifications under the same license, etc.
6. **Anonymous online activity is truly anonymous** - IP addresses, metadata, and digital footprints can often be traced back to individuals.
7. **AI and algorithms are unbiased** - Algorithms can reflect and amplify the biases present in their training data or design.
8. **Only hackers cause data breaches** - Human error (weak passwords, clicking phishing links, misconfigured systems) is a leading cause of data breaches.
