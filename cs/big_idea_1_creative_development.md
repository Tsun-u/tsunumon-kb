# Big Idea 1: Creative Development (CRD)

**Exam Weight:** 10-13%

## Overview

Computing is a creative discipline that enables innovation across many fields. The creative development process for computing innovations includes investigating, designing, prototyping, and testing. Collaboration and iteration are essential to effective development.

## Enduring Understandings and Learning Objectives

### CRD-1: Incorporating multiple perspectives through collaboration improves computing innovations as they are developed

#### CRD-1.A: Explain how computing innovations are improved through collaboration
- **Essential Knowledge:**
  - A computing innovation includes a program as an integral part of its function
  - A computing innovation can be physical (e.g., self-driving car), non-physical computing software (e.g., picture editing), or a non-physical computing concept (e.g., e-commerce)
  - Effective collaboration produces innovations that reflect the diversity of talents and perspectives of those who designed it
  - Collaboration can include pair programming, team programming, and peer review
  - Consultation and communication with users is important in developing innovations

#### CRD-1.B: Explain how computing innovations are developed by groups of people
- **Essential Knowledge:**
  - Online tools support collaboration (version control, shared documents, communication platforms)
  - Collaboration facilitates multiple perspectives in developing ideas for solving problems
  - Collaboration can help avoid bias in the development of computing innovations

#### CRD-1.C: Demonstrate effective interpersonal skills during collaboration
- **Essential Knowledge:**
  - Effective collaboration requires communication, consensus building, conflict resolution, and negotiation
  - Collaboration is important for both the development and the understanding of computing innovations

### CRD-2: Developers create and innovate using an iterative design process

#### CRD-2.A: Describe the purpose of a computing innovation
- **Essential Knowledge:**
  - The purpose of computing innovations is to solve problems or to pursue interests through creative expression
  - An understanding of the purpose of a computing innovation provides developers with an improved ability to develop that innovation

#### CRD-2.B: Explain how a program or code segment functions
- **Essential Knowledge:**
  - A program is a collection of program statements that performs a specific task when run by a computer (also referred to as software)
  - A code segment is a collection of program statements that is part of a program
  - The behavior of a program is how it functions during execution and is often described by how a user interacts with it

#### CRD-2.C: Identify input(s) to a program
- **Essential Knowledge:**
  - Program inputs are data sent to a computer for processing by a program
  - Input can come from a user or other programs
  - An event is associated with an action and supplies input data to a program

#### CRD-2.D: Identify output(s) produced by a program
- **Essential Knowledge:**
  - Program outputs are any data sent from a program to a device
  - Program output can include visual, auditory, haptic, or text outputs
  - An event is associated with an action and can produce output

#### CRD-2.E: Develop a program using a development process
- **Essential Knowledge:**
  - A development process can be ordered and intentional, or exploratory in nature
  - Common phases: investigating, designing, prototyping, testing
  - Requires iteration: revising and improving based on feedback and testing
  - A development process that is incremental allows programmers to implement and test small portions

#### CRD-2.F: Design a program and its user interface
- **Essential Knowledge:**
  - The design of a program incorporates investigation to determine its requirements
  - A program's specification defines the requirements for the program
  - In designing a program, the user interface should be intuitive

#### CRD-2.G: Describe the purpose of a code segment or program by writing documentation
- **Essential Knowledge:**
  - Program documentation helps programmers develop and maintain correct programs
  - Documentation includes descriptions of code segments, lists of contributors, references, and revision history
  - Comments are a form of program documentation written in the programming language

#### CRD-2.H: Acknowledge code segments used from other sources
- **Essential Knowledge:**
  - It is important to cite all sources of code, algorithms, and ideas
  - Using code developed by others or libraries as part of a program is common in software development

## Key Concepts and Terminology

| Term | Definition |
|------|-----------|
| Computing innovation | A creation that includes a program as an integral part |
| Iterative development | Repeatedly refining a solution through testing and feedback |
| Incremental development | Building a program in small, testable pieces |
| Pair programming | Two programmers working at one computer |
| Program documentation | Written descriptions that explain code |
| User interface (UI) | The means by which a user interacts with a program |
| Specification | A detailed description of program requirements |
| Prototype | An early model or version of a program |

## Common Misconceptions

1. **Programming is solitary work** - Modern software development is highly collaborative. Even "solo" projects use libraries written by others.
2. **Programs are written once and done** - Software development is iterative; programs are continuously revised, tested, and improved.
3. **Only the final product matters** - The development process (planning, testing, documenting) is just as important as the finished program.
4. **Copying code is always wrong** - Using and adapting code from libraries or open-source projects is standard practice, as long as it's properly attributed and licensed.
5. **Documentation is optional** - Good documentation is essential for maintaining and collaborating on code.
